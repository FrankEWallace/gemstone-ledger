---
title: Roles & Permissions
description: Two-tier access control — organization roles plus per-site roles.
---

Access control works on two levels: every user has one **organization role**, and separately a **role on each site** they've been given access to. A user with no role on a site cannot see that site at all.

## Organization roles

| Role | Description |
|---|---|
| **Owner** | Full control — sites, members, org settings, module configuration. The person who registers the organization becomes its owner. |
| **Admin** | Same day-to-day powers as owner: create sites, manage members and roles. |
| **Member** | Default for invited users. Access is determined entirely by their per-site roles. |

<Warning>
  An organization must always have at least one owner. The database blocks demoting or deleting the last owner.
</Warning>

## Site roles

| Role | Description |
|---|---|
| **Admin** | Full access to everything in the site, including inviting users to it |
| **Site manager** | Manages operations and can invite users to the site |
| **Worker** | Records day-to-day data (transactions, production, incidents) |
| **Viewer** | Read-only |

Site creators automatically get the site **admin** role, so a new site is immediately visible and manageable by whoever created it.

## Where enforcement happens

Permissions are enforced in the database with Postgres Row Level Security — every query is filtered to the sites the user has a role on, and privileged operations (creating sites, listing org members, inviting users) go through server-side functions that re-check the caller's role. The UI hides what you can't do, but the UI is never the only gate.

## Inviting users

Site admins and site managers can invite users from the **Team** page. The invite email is sent by a server-side function that first verifies the inviter's role on the target site; the recipient follows the link, sets a password, and lands in the org as a **member** with the site role the inviter chose.

## The Roles page

**Management → Roles** lists every member of your organization with their org role and all their site roles in one place, so owners/admins can audit who has access to what.
